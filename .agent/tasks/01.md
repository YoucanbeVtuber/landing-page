# LIVINGCEl 랜딩 페이지 요구사항 (Coding Agent Spec)

## 0) 목표

* 서비스명: **LIVINGCEl** (추후 변경 가능)
* 목적: Live2D 아바타 아티스트 대상 **제품 소개 랜딩 페이지** 제작
* 핵심 가치: `업로드 → AI 자동 레이어 분할/인페인팅 → 아티스트 튜닝 → 리깅 가능한 PSD 다운로드`
* 디자인 레퍼런스: **Apple 제품 소개 페이지 스타일**

  * 라이트 테마, 넓은 여백, 큰 타이포, 정돈된 레이아웃
  * 메인 포인트: **보라색 그라데이션**, “AI 서비스 느낌”(미묘한 글로우/블러 허용)

## 1) 추천 스택 (기본 전제)

* Next.js (App Router) + TypeScript
* Tailwind CSS
* Framer Motion (스크롤 progress 기반 애니메이션)
* 폼 제출: 더미(백엔드 연결은 추후)

## 2) 에셋/파일 구조 (Source of Truth)

* `public/hero/assets/cropped.png`  ✅ Hero 초기 원본으로 사용
* `public/hero/assets/original.png` (미사용, 필요 시 예비)
* `public/hero/layers/*.png` (레이어 분해 연출에 사용)

### 레이어 PNG 공통 특성

* 모든 레이어 png는 동일 해상도/좌표 정렬/알파 배경
* 같은 위치에 겹치면 합성된 캐릭터처럼 보인다

## 3) 페이지 섹션 구성

1. Hero (원본 → 3D 레이어 분해 + 후킹 카피)
2. Use Case (스크롤 기반 단계 연출: 업로드→분할→수정→재생성→내보내기)
3. Pre-register CTA (사전예약 버튼 → 연락처 입력 폼)

# 4) Hero 섹션 요구사항: “원본 → 3D 레이어 분해” 스크롤 연출

## 4.1 목표 연출

* 초기(progress 낮음): `cropped.png` 단일 원본처럼 보이게
* 스크롤 진행(progress 증가): `layers/*.png`가 **z축으로 분해(translateZ)**되어 “레이어가 분리됨”이 명확히 보이게
* 복잡한 3D/물리/Three.js 불필요. **CSS 3D transform + Framer Motion**으로 충분.

## 4.2 렌더링 규칙(필수)

* Hero 이미지 영역은 `position: relative`
* 레이어는 모두 같은 좌표에 `position: absolute; left:0; top:0;`로 겹친다
* 부모 컨테이너에:

  * `perspective: 800~1200px` 중 적절히 설정
  * `transform-style: preserve-3d`
* 각 레이어에:

  * `will-change: transform, opacity`

## 4.3 스크롤 progress 정의

* Hero 섹션 내부에서 스크롤 progress를 0~1로 정규화하여 사용(Framer Motion 권장)
* progress 구간 동작(권장):

  * `0.00 ~ 0.15`: 원본(cropped)만 보임 (레이어 스택 거의 숨김)
  * `0.15 ~ 0.30`: 원본 ↔ 레이어 스택 crossfade
  * `0.30 ~ 1.00`: 레이어 분해(핵심 구간)

## 4.4 원본 ↔ 레이어 전환(필수)

* `cropped.png`는 progress에 따라 서서히 opacity 감소
* 레이어 스택은 progress에 따라 서서히 opacity 증가
* crossfade로 전환이 자연스럽게 진행되도록 한다

## 4.5 레이어 순서: 코드 하드코딩 배열(필수)

* manifest는 사용하지 않는다.
* 아래 배열을 렌더 순서(뒤→앞) 기본값으로 사용하고, 추후 사용자가 수정 가능하도록 한다.

### Layer Render Order (back -> front) — Initial Default

```ts
export const HERO_LAYER_ORDER = [
  "base.png",
  "body.png",

  // face base
  "face_skin_base.png",
  "whole_face.png",

  // ears (behind hair by default)
  "face_ear_l.png",
  "face_ear_r.png",

  // eyebrows / lashes
  "eyebrow_l.png",
  "eyebrow_r.png",
  "eyelash_l.png",
  "eyelash_r.png",

  // eyes: white -> iris -> pupil -> highlight
  "eye_white_l.png",
  "eye_white_r.png",
  "eye_iris_l.png",
  "eye_iris_r.png",
  "eye_pupil_l.png",
  "eye_pupil_r.png",
  "eye_highlight_l.png",
  "eye_highlight_r.png",

  // mouth interior -> tongue -> teeth -> lips (adjust as needed)
  "mouth_inner.png",
  "tongue.png",
  "teeth_lower.png",
  "teeth_upper.png",
  "mouth_lips.png",

  // hair: back -> whole -> front (user may refine)
  "hair_back.png",
  "whole_hair.png",
  "hair_front.png",
] as const;
```

> 구현 시, 실제 경로는 `/hero/layers/${filename}`로 구성.

## 4.6 레이어 분해(depth) 애니메이션 규칙(권장)

* 기본 방식:
“폭발도(Exploded View)” 스타일로, 측면에서 바라보는 느낌의 3D 가로 분해”를 구현한다.

참고 방향성:
정면이 아니라 약간 옆에서 보는 시점
레이어가 두께를 가진 판처럼 보임
가로 방향으로 벌어짐
동시에 약간 전면으로 돌출

* 추가로 깊이감 강화:

- 앞쪽 레이어일수록 약간 더 밝게 보이게 하거나
- 미세한 그림자(soft drop shadow)로 레이어 분리 강조
- 레이어 간 미세한 z-offset 차등 적용

단, 과도한 회전/왜곡 금지.
미니멀하고 정제된 Apple 스타일 유지.

# 6) Use Case 섹션(스크롤 기반 단계 연출)

## 6.1 목적

* 사용자가 “서비스가 어떤 프로세스로 진행되는지” 이해하면 충분
* 실제 제품 UI를 그대로 재현할 필요 없음
* 카드/패널(정보 박스 형태의 가짜 UI)로 과정 연출

## 6.2 단계(고정)

1. 메인 페이지에서 이미지 업로드
2. “자동 분할” 버튼 클릭 → 레이어 분할 결과 표시
3. 마음에 들지 않는 레이어(머리카락)에서 “수정” 클릭
4. 수정 요청 입력: `앞머리만 분리해줘`
5. 재생성 결과 확인
6. “PSD 내보내기” 클릭 → 저장/완료 피드백

## 6.3 연출 규칙

* 스크롤 progress 구간별로:

  * 카드/패널이 등장(페이드/슬라이드)
  * 다음 단계로 전환될 때 이동/퇴장
* “클릭”은 실제 클릭 인터랙션이 아니라 **스크롤에 의해 클릭된 것처럼 보이게** 연출 가능

# 7) Pre-register CTA + 폼

## 7.1 동작(필수)

* 사전예약 버튼 표시
* 버튼 클릭 시 입력폼 표시
* 연락처 유형 선택: `이메일` 또는 `전화번호` 중 택 1
* 제출은 더미 처리(토스트/메시지). API 연결은 추후.

## 7.2 전화번호 입력 규칙(필수)

* 국가코드 없음(+82 없음)
* 숫자만 입력 받음(내부 상태 digits-only)
* 표시 형식에 자동으로 `-` 삽입/삭제

  * 예: `01012345678` → `010-1234-5678`
  * 10자리면 `010-123-4567` 형태
* 입력 중에도 자동 formatting이 적용되며, backspace 시 하이픈도 자연스럽게 갱신된다.

# 8) 구현 체크리스트(완료 조건)

* Hero: `cropped.png`에서 시작 → 스크롤에 따라 레이어 분해가 눈에 띄게 진행됨
* 레이어 렌더 순서 배열이 코드에 존재하며 수정 가능
* Use Case: 스크롤로 단계 전환이 시각적으로 이해 가능
* Pre-register: CTA → 폼 노출 → 이메일/전화 택1 → 전화 자동 하이픈 포맷 동작
* 전체: 라이트 테마 + 보라 그라데이션 + Apple식 타이포/여백 느낌 유지
